What is KnowHows?

KnowHows is a Slack application that tracks GitHub repositories and allows users to search for people who have knowledge on certain parts of code. By invoking the /knowhows command in a Slack channel, users can query KnowHows on an API term or a general software concept. KnowHows then produces a ranked list of tracked users that have the most experience with that query.

How does KnowHows work?

KnowHows uses a unique combination of a Solr search engine and the srcML source code parser to provide rankings for API terms. When a repository from GitHub is requested to be tracked, each commit is processed through srcML and correlated with fully patched files to identify relevant API terms. The terms and commit messages are then inserted into a Solr core that can be queried from the Slack application. Solr provides the heavy lifting by ranking individuals most heavily associated with that term or concept.

What gets tracked in a repository?

Currently, KnowHows only tracks Java (.java), C (.c), C++ (.cpp), and C# (.cs) source code commits. When a repository is tracked by you, only your authored commits are retrieved.

What kind of data do you store?

We does not store the full source code from repositories. We do however store the following possibly sensitive information:

  • Commit hashes
  • Commit messages
  • Filenames
  • Raw file URLs
  • Isolated terms from parsed code
  • GitHub access tokens

When a repository is untracked, all data is immediately queued for removal.
